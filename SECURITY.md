# Security Policy

## Supported Versions

We actively monitor and support security fixes for the following versions:

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

We take the security of DayTree very seriously. If you find a security vulnerability, please do not report it publicly via GitHub issues. Instead, report it responsibly by contacting the project maintainers directly or emailing security reports to the email listed on the owner's profile.

When reporting a vulnerability, please include:
* A detailed description of the issue.
* Steps to reproduce the vulnerability (proof-of-concept scripts or screenshots are highly appreciated).
* Potential impact of the issue.

We will acknowledge your report within 48 hours and work with you to patch and disclose the issue responsibly.
